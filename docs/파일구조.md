# DeepMetria 주요 파일 구조

> 실제 소스(`clean-base` 브랜치) 기준 정리. 계층형(레이어드) 구조이며, `핵심설계.md`의 AI 오케스트레이터 모델을 코드로 구현한 것이다.
> 의존 방향: **UI / App → Domain → Infra / Parsers / Render** (상위가 하위에 의존, 역방향 금지)

---

## 1. 한눈에 보기

```
src/
├─ App/        MFC 앱·문서·뷰·메인프레임 (진입점, 사용자 상호작용)
├─ UI/         보조 창·대화상자 (질문 입력, 요약/서식 패널, 설정, 내보내기)
├─ Domain/     핵심 도메인 — 분석 흐름·도구·대시보드·데이터 모델
├─ Infra/      외부 연동 — LLM Router/프로바이더, HTTP, 키 저장
├─ Parsers/    파일 파서 — CSV/Excel/JSON → DataSource
├─ Render/     GDI+ 차트 렌더링 (차트 유형별 전략)
├─ Common/     공용 — 윈도우 메시지 정의(WM_USER 등)
├─ Pch/        미리 컴파일 헤더(stdafx)·타깃 버전
└─ Resource/   .rc 리소스·아이콘·resource.h
```

---

## 1-A. 핵심 책임 요약 (꼭 알아야 할 파일 한 줄)

> 전체 80여 파일 중 **흐름을 이해하는 데 필수인 핵심만** 추렸다.

| 파일 | 핵심 책임 (한 줄) |
|------|-------------------|
| `Domain/AnalysisFlow` | **오케스트레이터** — 질문 1건을 계획→도구 선택→호출→결과까지 지휘 |
| `Domain/AnalysisTool` | **검증된 계산 함수 세트** + LLM 노출용 화이트리스트(`Catalog()`) |
| `Infra/LLMRouter` | **멀티 LLM 추상화** — 자연어→도구 호출 계획(`Plan()`), 오프라인 폴백 |
| `Infra/ILLMProvider` (+ Claude/OpenAI/Gemini) | 프로바이더별 포맷 차이를 흡수하는 어댑터 |
| `Infra/SecretsStore` | API 키 DPAPI 암호화 저장/복호화 |
| `Render/ChartRenderer` (+ *ChartStrategy) | 분석 결과를 GDI+ 차트로 렌더 (유형별 전략) |
| `Parsers/ParserFactory` (+ Csv/Xlsx/Json) | 파일 → `DataSource` 변환 |
| `App/DeepMetriaView` | 사용자 입력 수신 + 대시보드 렌더 표면 |
| `Common/Messages` | `WM_USER` — 워커 스레드 → UI 스레드 갱신 |

**한 문장 요약**: `DeepMetriaView`가 받은 질문을 `AnalysisFlow`가 지휘해 `LLMRouter`로 *판단*하고 `AnalysisTool`로 *계산*한 뒤, `ChartRenderer`로 그려 `WM_USER`로 화면을 갱신한다.

## 4. 핵심 흐름과 파일 매핑

**질문 → 시각화** (FR-03~05):
```
DeepMetriaView (입력)
  → AnalysisFlow (오케스트레이터)
      → LLMRouter.Plan() → ClaudeProvider 등 → HttpClient(WinHTTP)
      → AnalysisTool.<선택된 함수>() (검증된 계산)
  → ChartRenderer → *ChartStrategy (GDI+)
  → Common/Messages(WM_USER) → DeepMetriaView 갱신
```

설계 핵심이 코드에 그대로 드러남:
- **판단(AI)** = `LLMRouter` + `AnalysisFlow` (도구 이름·인자만 결정)
- **계산(검증된 코드)** = `AnalysisTool` (화이트리스트 `Catalog()` 내 함수만 실행)
- **추상화** = `ILLMProvider` 인터페이스 + 3개 프로바이더
- **보안** = `SecretsStore`(DPAPI)
- **반응성** = `Common/Messages`(WM_USER) + 워커 스레드

> 상세 설계 근거는 `docs/핵심설계.md`, 다이어그램은 `docs/diagrams/` 참조.
