```
|                  |    DataSummaryView          |

|                  |    DashboardView            |
|                  |    (우하단: 차트)             |
+------------------+-----------------------------+
```

## 2. QueryInputView (좌측 — 요청 창)

### 사전 조건
- 앱 정상 실행

### 2.1 초기 상태

| ID | 테스트 항목 | 조작 | 기대 결과 | Pass/Fail |
|----|-----------|------|----------|-----------|
| Q-01 | 컨트롤 표시 | 앱 실행 | 텍스트 입력란 + "분석" 버튼 + 상태텍스트 + 프로그레스바 표시 | |
| Q-02 | 초기 상태 텍스트 | 앱 실행 | "질문을 입력하고 분석 버튼을 누르세요." 표시 | |
| Q-03 | 프로그레스바 초기값 | 앱 실행 | 0% 상태 | |
| Q-04 | 분석 버튼 활성 | 앱 실행 | 버튼 클릭 가능 (enabled) | |

### 2.2 파일 로드 전 분석 시도

| ID | 테스트 항목 | 조작 | 기대 결과 | Pass/Fail |
|----|-----------|------|----------|-----------|
| Q-05 | 빈 질문 제출 | 입력 없이 "분석" 클릭 | "질문을 입력해주세요." 메시지박스 (경고 아이콘) | |
| Q-06 | 파일 미로드 질문 | 질문 입력 후 "분석" 클릭 | "데이터 파일을 먼저 불러오세요." 메시지박스 (경고 아이콘) | |

### 2.3 파일 로드 후

| ID | 테스트 항목 | 조작 | 기대 결과 | Pass/Fail |
|----|-----------|------|----------|-----------|
| Q-07 | 상태 텍스트 갱신 | CSV 파일 열기 | "데이터 로드 완료. 질문을 입력하세요." 표시 | |
| Q-08 | 질문 입력 | 텍스트 입력란에 한글 입력 | 정상 입력, 한글 깨짐 없음 | |
| Q-09 | 분석 시작 (버튼) | 질문 입력 → "분석" 클릭 | 상태: "분석 중...", 프로그레스바 진행, 버튼 비활성화 | |
| Q-10 | 분석 시작 (Ctrl+Enter) | 질문 입력 → Ctrl+Enter | Q-09와 동일하게 분석 시작 | |
| Q-11 | 분석 중 입력 차단 | 분석 진행 중 | 입력란 읽기전용, 버튼 비활성화 | |
| Q-12 | 프로그레스 단계 표시 | 분석 진행 중 관찰 | 25% → 50% → 75% → 100% 단계별 진행 | |

### 2.4 분석 완료/실패

| ID | 테스트 항목 | 조작 | 기대 결과 | Pass/Fail |
|----|-----------|------|----------|-----------|
| Q-13 | 분석 성공 | 분석 완료 대기 | 상태: "분석 완료.", 프로그레스 100%, 버튼 재활성화, 입력란 편집 가능 | |
| Q-14 | 분석 실패 | LLM 오류 발생 시 | 오류 메시지박스 (에러코드 + 상세), 버튼 재활성화 | |
| Q-15 | 실패 후 재시도 | 분석 실패 후 새 질문 입력 → 분석 | 정상적으로 새 분석 시작 (상태 초기화됨) | |
| Q-16 | 연속 질문 | 분석 성공 후 새 질문 → 분석 | 새 분석 정상 시작, 차트 카드 추가됨 | |

## 스크린샷 캡처 방법 (pywinauto)

테스트 중 스크린샷은 pywinauto 스크립트로 자동 캡처합니다.

### 사전 설치

```bash
pip install pywinauto Pillow
```

### 캡처 스크립트

```python
from pywinauto import Application
import datetime
import os

# 앱 연결 (이미 실행 중인 경우)
app = Application(backend="win32").connect(title_re=".*DeepMetria.*")
win = app.window(title_re=".*DeepMetria.*")

# 저장 경로
save_dir = ".claude/temp"
os.makedirs(save_dir, exist_ok=True)

def capture(name):
    """전체 윈도우 캡처"""
    ts = datetime.datetime.now().strftime("%H%M%S")
    path = f"{save_dir}/test_{name}_{ts}.png"
    win.capture_as_image().save(path)
    print(f"saved: {path}")
    return path

def capture_control(class_name, name):
    """특정 컨트롤만 캡처"""
    ts = datetime.datetime.now().strftime("%H%M%S")
    path = f"{save_dir}/test_{name}_{ts}.png"
    ctrl = win.child_window(class_name=class_name)
    ctrl.capture_as_image().save(path)
    print(f"saved: {path}")
    return path

# 사용 예시
capture("L-02_layout")                           # 전체 레이아웃
capture_control("SysListView32", "D-04_summary")  # DataSummaryView만
```

### 테스트 시 활용

1. 앱 실행 후 위 스크립트를 Python으로 실행
2. 테스트 항목별로 `capture("테스트ID_설명")` 호출
3. `.claude/temp/` 에 저장된 이미지 경로를 Claude에게 전달
4. Claude가 이미지를 읽고 Pass/Fail 판정
