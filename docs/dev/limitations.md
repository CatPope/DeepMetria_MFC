## 1. 플랫폼 제한사항

### Windows 전용

- **지원 OS**: Windows 10 (20H1, 빌드 19041) 이상, x64만 지원
- **미지원**: macOS, Linux, ARM64 Windows
- **영향**: 크로스 플랫폼 사용 불가. macOS/Linux 사용자는 별도 빌드 필요
- **이유**: MFC(Microsoft Foundation Classes)는 Windows 전용 프레임워크

### 32비트(x86) 미지원

- **현황**: x64 빌드만 제공
- **영향**: 구형 32비트 Windows 머신에서 실행 불가
- **범위**: Windows 10 x64 기준 영향 사용자 < 1%

### 관리자 권한 불필요 (일반 사용자 설치)

- 기본 설치 경로: `%ProgramFiles%\DeepMetria\` (UAC 프롬프트 1회)
- 이후 실행은 일반 사용자 권한으로 동작
- DB 및 설정은 `%APPDATA%\DeepMetria\`에 저장 (권한 문제 없음)

### 오프라인 시 LLM 기능 불가

- **현황**: LLM 분석(AI 질문 → 시각화)은 외부 LLM API 호출 필요
- **오프라인 동작 가능**: 파일 로드, 기존 대시보드 조회, 시각화 편집
- **오프라인 불가**: AI 자연어 분석 기능
- **완화책**: 이전 분석 결과는 로컬 DB에 저장되어 오프라인에서도 조회 가능

## 2. 기능 제한사항

### 지원 파일 형식

- **지원**: CSV, Excel (.xlsx, .xls), JSON
- **미지원**: Parquet, HDF5, XML, YAML, SQLite DB 직접 연결
- **파일 크기 한도**: 기본 200MB (app_settings에서 조정 가능)
- **영향**: 큰 데이터셋은 분할 후 로드 필요

### 데이터 영속성 (로컬 전용)

- **현황**: 모든 데이터는 로컬 SQLite에만 저장
- **미지원**: 클라우드 동기화, 다중 기기 간 데이터 공유
- **백업**: 사용자가 `%APPDATA%\DeepMetria\deepmetria.db` 수동 백업 필요
- **계획**: post-MVP에서 클라우드 백업 옵션 추가 검토

### 단일 사용자 환경

- **현황**: 단일 Windows 사용자 계정 기반 (다중 사용자 전환 없음)
- **미지원**: 사용자별 데이터 분리, 권한 관리
- **영향**: 공용 PC 환경에서 데이터 격리 불가

### LLM 의존성 및 비결정성

- **특성**: AI 분석 결과는 LLM 제공자의 응답에 따라 달라짐
- **문제**: 동일 데이터/질문에 대해 매번 다른 시각화 또는 분석이 생성될 수 있음
- **완화책**: 분석 이력 저장으로 이전 결과 재사용 가능

### 분석 도구 고정

- **범위**: MVP에서 제공하는 내장 분석 파이프라인 내에서만 동작
- **미지원**: 사용자 정의 C++ 플러그인, Python 스크립트 실행
- **계획**: post-MVP에서 플러그인 인터페이스 검토

### 동시 분석 제한

- **현황**: 한 번에 1개의 AI 분석만 실행 가능 (워커 스레드 1개)
- **영향**: 분석 중 다른 분석 요청은 큐에 대기

## 3. 성능 제한사항

### 대용량 파일 처리 시간

- **목표**: 100MB CSV 파일 로드 및 통계 계산 60초 이내
- **주의**: 1,000만 행 이상 파일은 메모리 부족 발생 가능
- **메모리 요구**: 파일 크기의 약 3배 (파싱 버퍼 포함) 예상
- **권장**: 파일 크기 50MB 이하에서 최적 성능

### AI 분석 응답 시간

- **목표**: 60초 이내 (LLM API 호출 시간 포함)
- **변수**: LLM 제공자 서버 부하, 네트워크 지연에 따라 변동
- **타임아웃**: 기본 60초, `app_settings.llm_timeout_sec`으로 조정 가능

### 대시보드 렌더링

- **목표**: 시각화 8개 이하 대시보드에서 2초 이내 초기 렌더링
- **주의**: GDI+ 기반 차트 렌더링으로 시각화 15개 이상 시 성능 저하
- **권장**: 대시보드당 시각화 10개 이하

### SQLite 동시성

- **현황**: WAL 모드 사용으로 읽기/쓰기 동시성 부분 지원
- **제한**: 다중 프로세스에서 동일 DB 파일 동시 접근 시 잠금 오류 가능
- **영향**: 앱을 여러 인스턴스로 동시 실행하지 말 것

## 4. 보안 제한사항

### API 키 저장 (Windows DPAPI)

- **현황**: LLM API 키를 Windows DPAPI로 암호화하여 SQLite에 저장
- **제한**: DPAPI는 동일 Windows 사용자 계정에서만 복호화 가능
- **위험**: 관리자 권한 공격자는 DPAPI를 우회할 수 있음
- **권장**: 신뢰할 수 없는 공용 PC에서 API 키 입력 금지

### 네트워크 통신 (LLM API)

- **현황**: libcurl + OpenSSL로 HTTPS 통신
- **제한**: 인증서 검증 기본 활성화 (비활성화 불가)
- **위험**: 기업 네트워크의 SSL 인터셉션 프록시 환경에서 연결 실패 가능
- **완화책**: 프록시 설정 지원 추가 예정 (post-MVP)

### 감사 로깅 부재 (Medium Severity)

- **현황**: 사용자 행동, 파일 로드, 분석 이력만 DB에 저장 (보안 감사 로그 없음)
- **영향**: 보안 사고 추적 어려움
- **일정**: post-MVP에서 로컬 감사 로그 추가 검토

### 파일 무결성 검사 없음 (Low Severity)

- **현황**: 로드된 파일의 SHA-256 해시 검증 미적용
- **영향**: 파일 로드 후 원본 파일 변조 감지 불가
- **완화책**: SQLite에 파일 경로 + 크기 + 수정 시간 저장으로 부분 감지

## 5. 알려진 이슈

### LLM 응답 타임아웃 시 UI 블로킹

- **증상**: LLM 타임아웃 발생 시 진행 표시기가 멈추고 취소 버튼이 느리게 반응
- **원인**: libcurl 타임아웃 처리와 MFC 메시지 루프 간 타이밍 이슈
- **회피책**: 분석 요청 시 앱이 멈춘 것처럼 보여도 60초 대기 후 자동 복구
- **개선**: 비동기 취소 로직 개선 예정

### 대용량 Excel 파일 (.xls) 파싱 실패

- **조건**: `.xls` 형식 (BIFF8) 파일 중 VBA 매크로 포함 시
- **증상**: 파싱 오류 메시지 표시, 데이터 로드 불가
- **회피책**: Excel에서 .xlsx 또는 .csv로 변환 후 로드
- **원인**: 현재 Excel 파서가 XLSB/VBA 매크로 처리 미지원

### 한글 파일 경로 처리

- **조건**: 파일 경로에 한글(UTF-8 다중바이트) 포함 시
- **증상**: 일부 환경에서 파일 열기 실패
- **회피책**: 파일을 영문 경로로 이동 후 로드
- **원인**: sqlite3, libcurl 일부 경로 처리 함수의 멀티바이트 처리 미흡
- **개선**: `wchar_t` 기반 경로 처리로 전면 교체 예정 (v1.1)

### 다크 모드 일부 UI 색상 오류

- **조건**: Windows 다크 모드 활성화 상태에서 앱 테마 라이트로 설정 시
- **증상**: 일부 MFC 컨트롤 배경색이 흰색으로 고정되지 않고 회색으로 표시
- **회피책**: 앱 설정에서 테마를 다크로 맞춤
- **개선**: MFC 다크 모드 지원 추가 예정 (v1.1)

## 6. Post-MVP 계획

### 기능 확장

| 항목 | 계획 시기 | 용도 |
|------|-----------|------|
| DB 직접 연결 (SQLite, PostgreSQL, MySQL) | post-MVP | 데이터 다이렉트 쿼리 |
| 클라우드 백업 (OneDrive/Google Drive) | post-MVP | 데이터 안전 보관 |
| 다중 분석 동시 실행 | post-MVP | 워커 스레드 풀 |
| 플러그인 인터페이스 | post-MVP | 사용자 정의 분석 확장 |
| macOS/Linux 포팅 (Qt 기반) | 장기 | 크로스 플랫폼 지원 |

### 보안 강화

- 프록시 설정 UI 및 libcurl 프록시 지원
- 감사 로그 (로컬 암호화 로그 파일)
- API 키 Windows Credential Manager 연동

### 성능 최적화

- GDI+ → Direct2D 렌더링 전환 (차트 성능 향상)
- 대용량 파일 스트리밍 파싱 (메모리 효율화)
- LLM 응답 로컬 캐싱
