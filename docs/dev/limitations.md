## 1. 기능 제한사항

### 지원 파일 형식
- **지원**: CSV, Excel (.xlsx, .xls), JSON 만 지원
- **미지원**: Parquet, HDF5, XML, YAML 등 다른 형식
- **영향**: 사용자는 데이터를 지원하는 형식으로 변환 후 업로드해야 함

### 파일 크기 제한
- **최대 업로드 크기**: 50MB
- **초과 시**: 업로드 실패, 오류 메시지 표시
- **권장**: 평균적인 분석 데이터 10~20MB 범위에서 최적 성능

### LLM 의존성 및 비결정성
- **특성**: AI 분석 결과는 LLM 제공자의 응답에 따라 달라짐
- **문제**: 동일한 데이터/질문에 대해 매번 다른 시각화 또는 분석이 생성될 수 있음
- **완화책**: 사용자가 여러 번 재시도하여 최적의 결과 선택 가능

### 실시간 협업 미지원
- **제한**: 동시에 여러 사용자가 같은 대시보드를 편집할 수 없음
- **현재**: 단일 사용자 세션 기반 (경합성 제어 없음)
- **충돌**: 두 사용자가 동시 편집 시 마지막 저장이 덮어쓰기함

### 분석 도구 고정
- **범위**: MVP에서 제공하는 AnalysisTool 세트 내에서만 도구 선택 가능
- **제한**: 사용자 정의 분석 함수 추가 불가
- **확장**: post-MVP에서 플러그인 방식 도구 확장 계획

## 2. 성능 제한사항

### 데이터 요약 생성 시간
- **목표**: 30초 이내
- **조건**: 일반적인 중소형 파일 (< 50MB, 행 수 < 1,000,000)
- **주의**: 매우 큰 파일이나 복잡한 스키마는 목표치 초과 가능

### 자연어 질문 → 시각화 생성
- **목표**: 60초 이내
- **포함**: LLM API 호출 시간 포함
- **변수**: LLM 제공자 응답 시간에 따라 실제 성능 변동
- **재시도**: LLM API 느림 시 자동 재시도 로직 제한적

### 대시보드 초기 로딩
- **목표**: 3초 이내
- **조건**: 브라우저 캐시 미적용
- **참고**: 시각화 개수가 많을수록 로딩 시간 증가

## 3. 보안 제한사항

### 감사 로깅 부재 (High Severity)
- **현황**: 사용자 행동, 데이터 접근, 시스템 이벤트 로깅 없음
- **영향**: 보안 사고 추적 불가능, 규제 준수 (GDPR, PCI-DSS) 어려움
- **위험**: 악의적 활동 탐지 및 사후 대응 불가
- **일정**: post-MVP에서 감사 로그 시스템 추가 예정

### 기본 시크릿 미변경 (High Severity)
- **현황**: 기본값 `APP_SECRET_KEY = "change-me"`
- **필수 조치**: 프로덕션 배포 전 반드시 강력한 값 (64자 이상 난수)으로 변경
- **영향**: 변경 미흡 시 토큰 위조 공격 위험
- **체크리스트**: `.env` 파일에 실제 값 설정 확인

### 속도 제한 (Rate Limiting) 없음 (High Severity)
- **현황**: 로그인 실패 시 속도 제한 없음
- **위험**: 비밀번호 브루트포스 공격 가능
- **영향**: 계정 탈취 위험
- **예정**: post-MVP에서 Rate Limiting 미들웨어 추가

### CORS 설정 미흡 (Medium Severity)
- **현황**: 모든 메서드, 모든 헤더 허용
- **권장**: 프로덕션 배포 시 특정 도메인만 허용하도록 설정
- **예시**: `CORS_ORIGINS=["https://yourdomain.com"]`

### 파일 무결성 검사 없음 (Medium Severity)
- **현황**: 업로드된 파일 해시(SHA-256) 검증 미적용
- **영향**: 저장 후 파일 변조 가능성
- **완화책**: UUID 기반 저장으로 파일명 위조 방어 (부분적)

## 4. 알려진 이슈

### LLM 응답 타임아웃
- **증상**: 분석 요청 중 LLM API 응답 없음 (> 60초)
- **현재 처리**: 사용자에게 오류 메시지 표시, 재시도 유도
- **개선**: 자동 재시도 로직 및 폴백(Fallback) 메커니즘 추가 예정

### 파일 업로드 실패 시 부분 저장
- **증상**: 업로드 실패 후에도 불완전한 파일 레코드가 DB에 남음
- **현재**: 다음 업로드 시 덮어쓰기되어 실제 문제 미미
- **개선**: 트랜잭션 롤백 강화 예정

### 시각화 렌더링 성능 저하
- **조건**: 대시보드에 시각화 10개 이상 추가 시
- **증상**: 드래그/리사이즈 반응 지연 (>500ms)
- **원인**: 프론트엔드 렌더링 최적화 미흡
- **회피책**: 시각화 개수 제한 (권장: 8개 이하)

## 5. Post-MVP 계획

### 기능 확장
| 항목 | 계획 시기 | 용도 |
|------|-----------|------|
| DB 직접 연결 (PostgreSQL, MySQL) | post-MVP | 데이터 다이렉트 쿼리 |
| 외부 API 연결 (Google Sheets, Salesforce) | post-MVP | API 기반 데이터 수집 |
| 과금/결제 시스템 | post-MVP | Free/Pro/Max 계획 |
| MCP/CLI/Skills 외부 공개 | post-MVP | 서드파티 개발 지원 |
| 온프레미스 배포 | post-MVP | 자체 인프라 지원 |

### 보안 강화
- 감사 로깅(Audit Logging) 추가
- Rate Limiting 미들웨어 구현
- 파일 무결성 검사 (SHA-256 해시)
- 비밀번호 정책 강화 (최소 길이, 복잡도)
- 세션 타임아웃 정책 수립

### 성능 최적화
- 프론트엔드 시각화 렌더링 개선 (가상화, 레이지 로딩)
- LLM 응답 캐싱 레이어 추가
- 데이터 요약 생성 병렬화
- 대시보드 페이지네이션 지원
