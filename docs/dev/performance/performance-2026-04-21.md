## 2. 테스트 실행 성능 기준선

| 범위 | 테스트 건수 | 전체 실행 시간 | 평균 시간/테스트 |
|------|-----------|----------------|-----------------|
| C++ 단위 테스트 (UT) | 150 | 5.42s | 36.1ms |
| 통합 테스트 (IT) | 59 | 12.08s | 204.7ms |
| MFC UI 단위 테스트 (UT) | 88 | 1.99s | 22.6ms |
| **전체 합계** | **297** | **19.49s** | **평균 65.6ms** |

**해석:**
- MFC UI 테스트는 GDI+ 렌더링 없이 로직만 테스트하므로 빠름
- 통합 테스트는 WinHTTP mock 초기화 오버헤드로 인해 UT보다 약 5.7배 느림
- C++ 단위 테스트는 메모리 연산 중심으로 CI 게이트로 적합
- 전체 테스트 스위트는 20초 이내에 완료되어 개발자 피드백 루프 친화적

## 3. NFR-01 성능 목표 대비 상태

| NFR-ID | 성능 요구사항 | 목표값 | 현재 상태 | 비고 |
|--------|------------|--------|---------|------|
| NFR-01a | 파일 로드 → 요약 생성 | ≤ 30초 | 미측정 | VS Profiler로 측정 필요 |
| NFR-01b | 자연어 질문 → 시각화 생성 | ≤ 60초 | 미측정 | LLM 추론 시간 포함 |
| NFR-01c | 대시보드 초기 렌더링 | ≤ 3초 | 미측정 | GDI+ 렌더링 최적화 필요 |
| NFR-01d | UI 응답 시간 (클릭 → 반응) | ≤ 200ms | 미측정 | VS Diagnostics로 측정 |
| NFR-01e | 장시간 실행 메모리 누수 | < 10MB 증가 | 미측정 | VS Memory Profiler 필요 |

**상태:** 모든 NFR 항목이 현재 단위/통합 테스트 범위를 벗어남. 배포 전 Visual Studio Profiler를 이용한 별도 성능 측정 실행 필수.

## 4. 측정 결과 및 분석

### 4.1 C++ 단위 테스트 성능 분석

**단위 테스트 (150 TC, 5.42s):**
- 평균 36.1ms/TC
- 외부 I/O 의존성 없음 (WinHTTP mock 사용)
- 비즈니스 로직 및 데이터 파싱 테스트에 적합
- **결론:** 정상 범위. CI/CD 게이트로 6초 이내 실행 기준 적용 가능

**통합 테스트 (59 TC, 12.08s):**
- 평균 204.7ms/TC
- WinHTTP mock 초기화 및 HTTP 세션 오버헤드 포함
- UT보다 5.7배 느림 (네트워크 레이어 mock 비용)
- **결론:** 수용 가능. mock 서버 초기화 캐싱 전략 검토 권장

### 4.2 MFC UI 단위 테스트 성능 분석

**UI 단위 테스트 (88 TC, 1.99s):**
- 평균 22.6ms/TC
- GDI+ 렌더링 없이 ViewModel 로직만 검증
- Visual Studio 테스트 프레임워크의 효율적인 로드 덕분
- **결론:** 우수. 개발 루프에서 실시간 피드백 가능

### 4.3 앱 런타임 성능 (미측정 — 계획)

**Visual Studio Profiler로 측정 예정:**
- CSV 파싱 처리 시간 (100행 / 10,000행 / 50MB)
- GDI+ 차트 렌더링 시간 (라인/바/테이블)
- 메모리 사용량 추이 (장시간 실행 1시간)
- UI 스레드 응답 시간 (메시지 펌프 지연)

### 4.4 기준선 해석

현재 단위/통합 테스트는 **C++ 로직 정확성**을 검증하는 단계이며, **실제 사용자 경험 성능(파일 파싱 속도, 렌더링, UI 응답성)**은 Visual Studio Profiler를 이용한 별도 측정이 필요합니다.

## 5. 권장 사항 및 다음 단계

### 5.1 즉시 실행 (개발 중)

1. **통합 테스트 최적화:**
   - WinHTTP mock 세션 초기화 캐싱 검토
   - Google Test 병렬 실행 고려 (`--gtest_parallel`)
   - 목표: IT 평균 시간을 100ms 이하로 단축

2. **GDI+ 렌더링 최적화:**
   - 더블 버퍼링(Double Buffering) 구현 확인
   - 불필요한 InvalidateRect 호출 최소화
   - 목표: 대시보드 초기 렌더링 ≤ 3초 달성

3. **성능 모니터링 기반 설정:**
   - 각 범위별 성능 회귀(regression) 테스트 추가
   - CI에서 기준선 대비 ±10% 초과 시 경고

### 5.2 배포 전 (Pre-deployment Gate)

1. **런타임 성능 측정 (Visual Studio Profiler):**
   - 파일 로드 → 요약 생성 전체 경과 시간 (NFR-01a)
   - LLM HTTP 응답 수신 → 시각화 완료 시간 (NFR-01b)
   - 목표: ≤ 30초 / ≤ 60초 달성 확인

2. **메모리 안전성 검증:**
   - VS Memory Profiler로 1시간 실행 후 누수 확인
   - 목표: 누수 없음, 메모리 증가 < 10MB

3. **Windows 버전별 성능 검증:**
   - Windows 10 22H2 / Windows 11 23H2에서 동일 기준 적용
   - DPI 96/120/144 환경별 렌더링 성능 비교

### 5.3 메트릭 추적

| 메트릭 | 측정 방법 | 검토 주기 |
|--------|---------|---------|
| 테스트 스위트 실행 시간 | CI 로그 (Google Test XML) | 매 빌드 |
| 파일 파싱 처리 시간 | VS Profiler 수동 실행 | 주 1회 |
| GDI+ 렌더링 시간 | VS Diagnostics | 주 1회 |
| 메모리 사용량 | VS Memory Profiler | 배포 전 |
| UI 응답 시간 | VS Diagnostics Tools | 배포 전 |

### 5.4 다음 성능 보고서 주기

- **개발 중:** 주 1회 (단위/통합 테스트 기준선 추적)
- **배포 전:** VS Profiler 측정 완료 후 별도 보고서 생성
- **배포 후:** 월 1회 (실제 사용 환경 성능 기반)

## 요약

DeepMetria MFC 앱의 **개발 환경 테스트 성능 기준선**이 확립되었습니다. C++ 단위/통합 테스트 및 MFC UI 테스트는 빠르고 안정적이지만(5.42s + 12.08s + 1.99s = 19.49s), 실제 사용자 경험(파일 파싱 속도, GDI+ 렌더링, UI 응답성, 메모리 안전성)에 대한 측정은 배포 전 Visual Studio Profiler 단계에서 실행됩니다. 현재 단계는 **C++ 로직 정확성 검증**에 초점이 있으며, **다음 단계(배포 전)에서 NFR-01 모든 목표를 대상으로 프로파일링을 수행**할 예정입니다.
